### Added
- Initial repository scaffold: `.gitignore`, `README.md`, `CHANGELOG.md`, `LICENSE`, `docs/ROADMAP.md` stub.
- Research phase 1: comparative survey of existing open-source backtesters under [`docs/research/0001-existing-backtesters.md`](docs/research/0001-existing-backtesters.md), with per-library detailed analyses in [`docs/research/sources/`](docs/research/sources/).
- Research phase 2: methodology canon synthesis under [`docs/research/0002-methodology.md`](docs/research/0002-methodology.md), with per-topic source analyses in [`docs/research/sources/methodology-*.md`](docs/research/sources/) covering AFML chs 11-15 on backtesting, the Bailey and Lopez de Prado backtest-overfitting papers (PSR, DSR, PBO, MinTRL), the Almgren-Chriss optimal-execution model and the square-root market-impact literature, the point-in-time data treatment across five PIT axes, and seven substantive practitioner postmortems.
- ADR 0001 ([`docs/decisions/0001-spec-critique.md`](docs/decisions/0001-spec-critique.md)): critique of the original project spec with a skeptical-reviewer pass and final decisions. Reframes the project as a teaching artifact with explicit non-goals; commits CPCV as the primary validation surface; commits the LdP chapter 14 scorecard as the default analytics; commits Polars end-to-end with `.to_pandas()` adapter; commits Sharadar SF1 ARQ + SEP + TICKERS as the v1 data inventory with documented gaps.
- ADR 0002 ([`docs/decisions/0002-roadmap-review.md`](docs/decisions/0002-roadmap-review.md)): M1 through M5 implementation milestone breakdown with a skeptical-reviewer pass and final decisions. Extends the v1 timeline from four weeks to ten weeks after the reviewer's honest-hours analysis (15-25 effective hours per week across DripWatch, Kalshi, undergraduate obligations). Cuts `zipline-reloaded` differential testing from v1 (moved to v1.1 backlog) and moves the kill-early gate from end of week 1 to end of week 2 on the SPY reconciliation. Locks acceptance criteria for each milestone including the Bailey-LdP 2014 numerical replication (DSR=0.971 within 1e-3) for M4 and the honest DSR reporting requirement for M5 (the strategy passes the milestone whether it clears DSR>=0.95 or fails it).
- ADR 0003 ([`docs/decisions/0003-architecture.md`](docs/decisions/0003-architecture.md)): core architecture sketch with skeptical-reviewer pass and 23 locked decisions. Protocol hierarchy across the six layers (data, signal, policy, execution, risk decomposition, analytics); single-process sequential BarLoop with multiprocess Runner for CPCV paths; trust boundaries enumerated (11 items); Pydantic restricted to adapter load, CLI/config, and user-facing render targets with attrs `slots=True frozen=True` for inner-loop types; CashFlow split from CorporateAction as two streams; permanent impact lives in the data source layer via `ImpactedPriceSource`; `PreTradeCostEstimator` and `FillCostComputer` split into separate protocols; `Signal.compute` returns `dict[AssetId, float]`; `MatchingEngine.submit` returns `list[Fill]`; Clock includes `is_market_open` and `next_bar`; `AssetId = NewType("AssetId", int)` with separate `IdentifierResolver`; LongOnlyPolicy at v1; determinism invariant documented. AlgoStack from bt dropped in favor of a single `Policy.target_positions` function (can return in v1.1).
- Pre-M1 methodology suite: [`docs/methodology/total_return_reconstruction.md`](docs/methodology/total_return_reconstruction.md), [`docs/methodology/dataset_versioning.md`](docs/methodology/dataset_versioning.md), [`docs/methodology/pydantic_polars_boundary.md`](docs/methodology/pydantic_polars_boundary.md), [`docs/methodology/determinism.md`](docs/methodology/determinism.md). Lock the SPY reconciliation reference (SPDR-published NAV TR from SSGA with explicit expense-ratio drag in the 5-bps budget); the Sharadar snapshot SHA256 manifest convention under `data/snapshots/manifest.toml`; the Pydantic-at-three-surfaces boundary with attrs everywhere else; and the determinism invariant with its five requirements and 11-item trust boundary table.
- Pre-M1 package scaffold: `src/pit_backtest/` layout per ADR 0003 with protocols stubbed (`Protocol` body `...`) and concrete classes raising `NotImplementedError("<milestone> deliverable")`. Includes `pyproject.toml` with pinned dependencies (polars==1.27.0, numpy==1.26.4, pydantic==2.9.2, attrs==24.2.0, pandas-market-calendars==4.4.1) per the determinism invariant; mypy strict configuration; pytest with `pytest-env` for `PYTHONHASHSEED=0`; ruff defaults. Minimal `tests/test_scaffold.py` verifies the package imports and the locked API contracts (attrs immutability on `PriceRecord`, `FillPriceModel` required on `Order`, `NoImpact` requires `unsuitable_for_deployment=True`, render-path enforcement raises on raw SR without PSR or DSR when the confidence tier is not `single_run_pre_specified`).

- **Pre-M2 design (ADR 0005)**: M2 cost-realism implementation plan with 18 locked decisions. Ran Plan + skeptical-reviewer subagents per the project rule (now formalized in [`memory/project_pitbacktest.md`](../../.claude/projects/C--Users-SamJD-OneDrive-Desktop-AI-Projects/memory/project_pitbacktest.md) working rule 2). Reviewer surfaced 5 Critical/High findings; all addressed before any M2 code:
  - Almgren 2005 formula labeled as `tcost_fraction` (decimal) at headline, `tcost_bps = tcost_fraction * 10_000` derived explicitly (fixes a unit-error trap that would have failed the headline acceptance criterion).
  - `SquareRootImpactMatchingEngine` raises `UnsupportedFillPriceModelError` for `FillPriceModel.VWAP` rather than silently substituting OHLC4 (preserves the `CloseFillMatchingEngine` no-silent-substitution discipline).
  - Perf-budget CI runs on synthetic data, labeled explicitly (the Sharadar CI gap is real and the original plan was silent on it).
  - Pre-trade vs fill-cost tolerance rewritten as `0.5 bps + 0.1 * |delta_mid|_bps` (the original `5 bps + 1 day sigma_D` was 125 bps for SPY, a non-test).
  - `/100` regression test targets `PerShareCommission` specifically (the historical backtrader bug class), with `BasisPointsCommission` as an additional test.
  - `ImpactedPriceSource` on Signal `pit_view` defaults OFF to avoid the mean-reversion feedback loop (queued as ADR 0007 with PR B).
  - M2 splits into 4 PRs (cost-model math; ImpactedPriceSource+matcher; sensitivity-band runner; perf-budget CI), each with its own reviewer pass.
  - Spins off ADR 0006 (acceptance criterion revision for FIM 2018 cross-check) with PR A and ADR 0007 (`ImpactedPriceSource` Signal policy) with PR B.
