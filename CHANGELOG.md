## [Unreleased]

### Added
- Initial repository scaffold: `.gitignore`, `README.md`, `CHANGELOG.md`, `LICENSE`, `docs/ROADMAP.md` stub.
- Research phase 1: comparative survey of existing open-source backtesters under [`docs/research/0001-existing-backtesters.md`](docs/research/0001-existing-backtesters.md), with per-library detailed analyses in [`docs/research/sources/`](docs/research/sources/).
- Research phase 2: methodology canon synthesis under [`docs/research/0002-methodology.md`](docs/research/0002-methodology.md), with per-topic source analyses in [`docs/research/sources/methodology-*.md`](docs/research/sources/) covering AFML chs 11-15 on backtesting, the Bailey and Lopez de Prado backtest-overfitting papers (PSR, DSR, PBO, MinTRL), the Almgren-Chriss optimal-execution model and the square-root market-impact literature, the point-in-time data treatment across five PIT axes, and seven substantive practitioner postmortems.
- ADR 0001 ([`docs/decisions/0001-spec-critique.md`](docs/decisions/0001-spec-critique.md)): critique of the original project spec with a skeptical-reviewer pass and final decisions. Reframes the project as a teaching artifact with explicit non-goals; commits CPCV as the primary validation surface; commits the LdP chapter 14 scorecard as the default analytics; commits Polars end-to-end with `.to_pandas()` adapter; commits Sharadar SF1 ARQ + SEP + TICKERS as the v1 data inventory with documented gaps.
- ADR 0002 ([`docs/decisions/0002-roadmap-review.md`](docs/decisions/0002-roadmap-review.md)): M1 through M5 implementation milestone breakdown with a skeptical-reviewer pass and final decisions. Extends the v1 timeline from four weeks to ten weeks after the reviewer's honest-hours analysis (15-25 effective hours per week across DripWatch, Kalshi, undergraduate obligations). Cuts `zipline-reloaded` differential testing from v1 (moved to v1.1 backlog) and moves the kill-early gate from end of week 1 to end of week 2 on the SPY reconciliation. Locks acceptance criteria for each milestone including the Bailey-LdP 2014 numerical replication (DSR=0.971 within 1e-3) for M4 and the honest DSR reporting requirement for M5 (the strategy passes the milestone whether it clears DSR>=0.95 or fails it).
- ADR 0003 ([`docs/decisions/0003-architecture.md`](docs/decisions/0003-architecture.md)): core architecture sketch with skeptical-reviewer pass and 23 locked decisions. Protocol hierarchy across the six layers (data, signal, policy, execution, risk decomposition, analytics); single-process sequential BarLoop with multiprocess Runner for CPCV paths; trust boundaries enumerated (11 items); Pydantic restricted to adapter load, CLI/config, and user-facing render targets with attrs `slots=True frozen=True` for inner-loop types; CashFlow split from CorporateAction as two streams; permanent impact lives in the data source layer via `ImpactedPriceSource`; `PreTradeCostEstimator` and `FillCostComputer` split into separate protocols; `Signal.compute` returns `dict[AssetId, float]`; `MatchingEngine.submit` returns `list[Fill]`; Clock includes `is_market_open` and `next_bar`; `AssetId = NewType("AssetId", int)` with separate `IdentifierResolver`; LongOnlyPolicy at v1; determinism invariant documented. AlgoStack from bt dropped in favor of a single `Policy.target_positions` function (can return in v1.1).
- Pre-M1 methodology suite: [`docs/methodology/total_return_reconstruction.md`](docs/methodology/total_return_reconstruction.md), [`docs/methodology/dataset_versioning.md`](docs/methodology/dataset_versioning.md), [`docs/methodology/pydantic_polars_boundary.md`](docs/methodology/pydantic_polars_boundary.md), [`docs/methodology/determinism.md`](docs/methodology/determinism.md). Lock the SPY reconciliation reference (SPDR-published NAV TR from SSGA with explicit expense-ratio drag in the 5-bps budget); the Sharadar snapshot SHA256 manifest convention under `data/snapshots/manifest.toml`; the Pydantic-at-three-surfaces boundary with attrs everywhere else; and the determinism invariant with its five requirements and 11-item trust boundary table.
- Pre-M1 package scaffold: `src/pit_backtest/` layout per ADR 0003 with protocols stubbed (`Protocol` body `...`) and concrete classes raising `NotImplementedError("<milestone> deliverable")`. Includes `pyproject.toml` with pinned dependencies (polars==1.27.0, numpy==1.26.4, pydantic==2.9.2, attrs==24.2.0, pandas-market-calendars==4.4.1) per the determinism invariant; mypy strict configuration; pytest with `pytest-env` for `PYTHONHASHSEED=0`; ruff defaults. Minimal `tests/test_scaffold.py` verifies the package imports and the locked API contracts (attrs immutability on `PriceRecord`, `FillPriceModel` required on `Order`, `NoImpact` requires `unsuitable_for_deployment=True`, render-path enforcement raises on raw SR without PSR or DSR when the confidence tier is not `single_run_pre_specified`).
- **M1 day 1**: `reconstruct_total_return(prices, dividends, start_dt, end_dt, expense_ratio_annual)` in `src/pit_backtest/data/adjustments.py`. Pure Polars implementation of the methodology doc's algebra; toy 3-day fixture from `total_return_reconstruction.md` Worked Example A passes exactly (TR[3] = 1.0352238806...). Companion `annualized_return` uses the 252-trading-day SSGA convention. 8 unit tests covering the toy fixture, no-dividend, dividend-outside-window, expense-ratio drag, annualization, and input-validation paths.
- **M1 day 1**: `load_manifest` and `verify_bundle` in `src/pit_backtest/data/sources/manifest.py`. Streaming SHA256 verifier (1 MiB chunks) per `dataset_versioning.md` schema; `SnapshotMismatchError` surfaces expected vs actual hashes and points at the refresh procedure. 11 unit tests against synthetic bundles.
- **M1 day 1**: `SharadarDataSource` in `src/pit_backtest/data/sources/sharadar.py`. Constructor verifies the bundle SHA256s before any read; `get_table(name)` returns cached lazy frames for SEP/ACTIONS/SF1/TICKERS/SP500; M1 convenience methods `read_sep_prices(ticker, ...)` and `read_actions_dividends(ticker, ...)` drive the SPY reconciliation without requiring the M3 `IdentifierResolver`. Per-row `PitDataSource` methods (`get_price`, `get_cash_flows`, etc.) remain `NotImplementedError` pending M3. 10 unit tests cover the construction-time SHA256 check, the ticker and date-range filters, the dividend-only filter on ACTIONS, the lazy-frame cache, the unknown-table error, and the end-to-end adapter -> reconstruction TR path against a synthetic SPY fixture.
- **M1 day 2**: `TestClock` in `src/pit_backtest/execution/clock.py` per ADR 0003 decision 7. Backed by pandas-market-calendars NYSE; trading-day cache loaded once at construction with a 14-day padding so `next_bar` lookups near the window edge succeed. `now`, `is_market_open`, `next_bar`, `advance_to`, plus the `trading_days()` accessor the BarLoop will iterate. Naive datetimes interpreted as America/New_York; aware datetimes converted. 16 unit tests covering NYSE holidays (Good Friday, Juneteenth, Independence Day), weekends, advance-to-non-trading-day-snaps-forward, and the strictly-after semantics of `next_bar`.
- **M1 day 2**: `configure_logging` and `get_logger` in `src/pit_backtest/utils/logging.py` per ADR 0002 decision 5. Structured `key=value` fields appended to every record via a custom formatter; values containing whitespace or `=` are repr'd for unambiguous grep parsing. Idempotent reconfigure (most-recent-level-wins). 7 tests.
- **M1 day 2**: `SSGASpyReference` in `src/pit_backtest/data/sources/ssga.py`. Loads the SPDR-published SPY snapshot (Performance + Distributions CSVs) under `data/snapshots/spy_ssga_<YYYY-MM-DD>/` with SHA256 verification. Lazy-cached `dividends()` and `performance()` Polars frames; `annualized_nav_tr_for_period(period)` returns the SSGA-published annualized NAV TR as a decimal. Companion `reconciliation_delta_bps(engine, ssga)` returns the engine-vs-SSGA delta in bps. 10 tests against synthetic Performance/Distributions CSVs.
- **M1 day 2**: `reconcile_spy` and `ReconciliationReport` in `src/pit_backtest/engine/spy_reconciliation.py`. Composes `SharadarDataSource` + `SSGASpyReference` + `reconstruct_total_return` into the M1 kill-early gate. Returns a frozen `ReconciliationReport` with `passes_kill_gate(tolerance_bps=5.0)` and `render_evidence_line()` for the PR description. `discover_latest_bundle(snapshots_root, prefix)` defaults to the most recent matching pull when callers do not pass an explicit bundle.
- **M1 day 2**: `tests/integration/test_spy_reconciliation.py`. Synthetic-fixture mode (CI-enabled) builds matching Sharadar + SSGA bundles with a hand-chosen 1.0005-per-day multiplier over 253 trading days, runs `reconcile_spy`, asserts delta < 1 bp. Real-snapshot mode (CI-skipped via `@pytest.mark.snapshot`) runs against `data/snapshots/sharadar_<YYYY-MM-DD>/` + `data/snapshots/spy_ssga_<YYYY-MM-DD>/`, asserts `|delta| <= 5 bps` over 2005-2024 (`@pytest.mark.kill_gate`). One-quarter preflight test on 2024 Q1 with 20 bps tolerance per `total_return_reconstruction.md`.
- **M1 day 2**: `examples/spy_buy_and_hold.py` CLI demo. Loads the Sharadar snapshot, prints annualized return; `--compare-to-ssga` runs the full reconciliation and prints the evidence line per `dataset_versioning.md`. Auto-discovers the latest bundle via `discover_latest_bundle` when bundle name is omitted. Exit code 0 on pass, 1 on kill-gate failure, 2 on missing snapshot.

### Changed
- Bumped Polars pin from yanked `1.27.0` to `1.41.1` (uv reported the prior pin as yanked from PyPI); committed `uv.lock` per ADR 0002 acceptance criterion 5. Pre-M1 correction; subsequent Polars bumps follow the determinism doc's reconciliation-delta-in-PR-description rule.
- `docs/methodology/dataset_versioning.md`: inventory table now lists Sharadar ACTIONS as the dividend source (the doc had been implicit about where dividends come from); snapshot path convention and example manifest updated to include `actions.parquet`.

### Changed
- README reframed from "production-grade event-driven backtester" to "U.S. equity daily-bar backtester built as a teaching artifact" per ADR 0001 final decisions. Added explicit non-goals section, design-pillar updates, performance budget, and v1 data inventory.
- ROADMAP updated through all three phase 3 ADRs: ADR 0001 (locked decisions carried forward), ADR 0002 (full M1 through M5 phase with ten-week timeline, acceptance criteria, v1.1 backlog), ADR 0003 (architecture decisions carried into the pre-M1 Monday work and the package scaffold).
