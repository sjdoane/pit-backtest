- **`SquareRootImpactMatchingEngine`** at `src/pit_backtest/execution/matching.py` per ADR 0009 lock #4 + #9. Supports `FillPriceModel.OPEN/CLOSE/ARRIVAL`. Raises `UnsupportedFillPriceModelError` for VWAP (v1.1 intraday data per ADR 0005 step 4) and NEXT_BAR_OPEN (M3 deferred-fill mechanism per ADR 0009 lock #4). Sign convention: buy lifts fill_price above arrival; sell lowers; `signed_perm` uses `fill_price` not `arrival` so OPEN/CLOSE/ARRIVAL agree on the dollar magnitude that hits the impact register. Decimal-at-boundary discipline preserved via `_to_boundary_decimal` (shared with PR A's cost-model module).
- **`MatchingEngine` Protocol extended with `on_bar_start(bar_dt: date) -> None`** per ADR 0009 lock #6. `CloseFillMatchingEngine` adds a no-op implementation so the M1 matcher continues to satisfy the extended Protocol. The BarLoop calls `matching_engine.on_bar_start(bar_dt)` unconditionally at the top of each per-bar iteration; the prior `hasattr` guard would have silently failed on a `on_bar_started` typo.
- **`MarketState` gains `prior_close: Decimal | None = None`** field per ADR 0009 lock #5 for `FillPriceModel.ARRIVAL` resolution. No `next_open` field is added because NEXT_BAR_OPEN is deferred to M3.

- **Hotfix: Sharadar adapter date filter regression + pandas pin.** Sam re-pulled Sharadar through 1993-01-22 to exercise the SI window after PR #17 merged; the new bundle reported "bundle has no SPY rows" on every window despite containing 7126 SPY rows from 1997-12-31 to 2026-04-30. Root cause: `pl.from_pandas` under the previously-broken `pandas==3.0.3` (a yanked-or-prerelease version transiently in `uv.lock`) accidentally returned a nullable-date dtype which polars surfaced as `pl.Date`; the cast-after-filter pattern in `read_sep_prices` and `read_actions_dividends` "worked" by coincidence under that wrong version. After pinning `pandas==2.2.3` (the correct, documented version that `nasdaq-data-link 1.0.4` was tested against), `pl.from_pandas` returns the standard `Datetime(time_unit='ns')` shape and exposes the latent cast-after-filter bug. Narrow-window queries within the representable range still worked, masking the bug from any test that did not exercise the wide-open coverage probe that `reconcile_spy_trailing` uses. Fix: cast `date` to `pl.Date` via `.with_columns()` BEFORE any filter in both `read_sep_prices` and `read_actions_dividends` (pl.Date has a wider representable range and accepts the wide-open literal). New regression test `test_date_range_filter_works_on_datetime_typed_input` in `tests/data/test_sharadar_adapter.py` uses an EXPLICIT `pl.Datetime(time_unit="ns")` override (bare `pl.Datetime` defaults to `time_unit="us"` which does NOT overflow at 2999-12-31 and would silently neutralize the test), asserts the on-disk dtype is `ns` to lock against future Polars default changes, and exercises both the narrow-window and wide-open coverage paths on read_sep_prices AND read_actions_dividends.

- **`SquareRootImpactCostModel`** implementing the Almgren 2005 Risk magazine v18 Section 3 formula `tcost_fraction = (1/2)*gamma*sigma_D*(Q/V_D)*(Theta/V_D)^(1/4) + eta*sigma_D*|Q/(V_D*T)|^beta` with default calibration eta=0.142, beta=0.6, gamma=0.314. Q convention: shares (not dollar notional). T=1.0 at v1 per ADR 0005 step 7 (one-fill-per-(asset, dt) for daily bars). Cost model is constructed once per backtest run and reused across all estimate/compute calls; market state is injected via `MarketStateLookup`.
- **`MarketStateLookup`** + **`MarketStateRow`** attrs-frozen wrappers around `dict[(AssetId, date), MarketStateRow]` with construction-time validation of V_D > 0, Theta > 0, sigma_D >= 0. Dict-of-tuples shape chosen over Polars frame for O(1) lookup in the ~500x/bar hot path; PR B's `Backtest.__init__` materializes the dict once from a Polars frame per ADR 0005 step 8.
- **`PerShareCommission.commission_for(shares, notional) -> Decimal`** and **`BasisPointsCommission.commission_for(shares, notional) -> Decimal`** implementations. Signed-notional convention: `shares` is signed at Order boundary; `notional = shares * fill_price` is signed; commission is always positive via `abs()`. `_BPS_TO_FRACTION_DIVISOR = Decimal("10000")` is exact (no float coercion).
- **`compute_rolling_adv(volume, window=20)` + `compute_rolling_daily_vol(returns, window=20)`** in `src/pit_backtest/data/rolling.py` via pure NumPy `np.convolve` per ADR 0005 step 8. `compute_rolling_daily_vol` uses the true standard deviation via the second-moment-minus-first-moment-squared identity, NOT RMS (the reviewer's Critical finding C3: RMS biases sigma upward by `mean(r)^2` for non-zero-mean return series). Alignment contract: output index `i` covers input bars `[i, i+window-1]`; lookahead-safe ADV at bar `t` uses `output[t-window]` on `volume[:t]` (the leak-test in `tests/data/test_rolling.py::test_rolling_adv_no_lookahead` demonstrates the off-by-one consequence).
